1. Design Philosophy
LalaKirana is not a product for the market. It is a tool built by a family, for a family. The design reflects this: warm, unhurried, deeply familiar — like the shop itself. Every decision pushes against the generic SaaS admin template.

Design Principles — What This UI Must Feel Like
•  Ledger-like — not a dashboard. The shop has 50 years of paper records. The UI should feel like their digital continuation.
•  Operator-first — the billing screen is used under pressure. Clarity and speed beat aesthetics when it counts.
•  Warm at rest, sharp in action — the idle UI is earthy and calm. Interactive states are high-contrast and immediate.
•  No generic blue — not a single default #1E40AF anywhere. Every color is earned from the palette.
•  Typography carries personality — Playfair Display headings feel like a hand-lettered signboard over the counter.
•  The sidebar is the spine of a register book — structured, authoritative, quietly marking your place.

8. Responsive Behaviour (PWA)
Phase 1 is desktop-first — the shop has one computer. However the PWA must not break if opened on a phone (for the owner checking remotely).

Breakpoint
Sidebar
Layout
≥ 1280px (desktop)
240px fixed, always visible
Full side-by-side layout
1024–1279px (laptop)
240px fixed
Tables may scroll horizontally
768–1023px (tablet)
Collapsed to 64px icon-only sidebar
Drawers become full-screen sheets
< 768px (mobile)
Hidden, accessible via hamburger menu
Single column, stacked cards

PWA manifest: theme_color = #2E1F26 (Caramel). background_color = #F5ECD7 (Parchment). This ensures the browser chrome matches the app on install.

9. Accessibility
Rule
Implementation
Colour contrast
Parchment on Caramel: 9.2:1 ✓   Parchment on Raisin: 4.8:1 ✓   Ink on Parchment: 18:1 ✓
Focus ring
2px Raisin outline, 2px offset — visible on all interactive elements
Touch targets
Minimum 44×44px — all buttons and nav items
Screen reader
All icons have aria-label; status badges have aria-live='polite'
Reduced motion
All CSS transitions wrapped in @media (prefers-reduced-motion: no-preference)
Keyboard nav
Full tab order; sidebar navigable with arrow keys; modals trap focus
Error messages
Associated with inputs via aria-describedby; not colour-only indicators
